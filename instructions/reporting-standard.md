# Reporting Standard

Use this file to keep reports consistent across target types.

## Mandatory sections

- Executive Summary
- Key Findings
- Scope
- Methodology
- Collector Execution Summary
- Confirmed Evidence
- Leads Requiring Validation
- Recommended Next Steps

## Writing rules

- Prefer meaning over raw pipeline chatter.
- State which collectors were executed and which query each collector used.
- Keep collector failures visible but separate from findings.
- Use target-specific sections when possible.
- Keep reference hubs separate from target-specific pivots.

## Confidence rules

- Confirmed evidence: directly returned by a collector or a reviewed source.
- Lead requiring validation: plausible pivot that needs manual confirmation.
- Reference pivot: useful next lookup, not evidence.
