- relire `README.md`
- garder la documentation bilingue coherente
- verifier que `.env` n'est pas versionne
- verifier que les sorties brutes ne sont pas poussees

- poser clairement les limites d'usage ethique et autorise
- expliquer ce qui est deja implemente vs la roadmap
- garder un README aligne visuellement avec le reste de ton GitHub lab
