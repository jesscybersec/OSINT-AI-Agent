# Canada OSINT Methodology

Use this skill when a target has Canadian context or when a Canada-specific profile is selected.

## Workflow

1. Confirm the Canadian angle.
   - Identify whether the target is tied to Canada by location, phone numbering, employer, registry context, or public-reference scope.
2. Prioritize Canadian sources before global noise.
   - Prefer Government of Canada, provincial, municipal, legal, directory, and registry pivots over generic web search.
3. Keep the reporting regional.
   - Record province, municipality, agency, registry, or directory context when a source is Canadian.
4. Separate record types.
   - Public record
   - Directory or index hit
   - News/reference hit
   - Derived lead requiring manual confirmation
5. End with regional next steps.
   - Recommend the next Canada-specific pivots instead of only global search engines.

## Reporting Rules

- Name the Canadian source family that informed the finding.
- Distinguish `Canadian public record` from `Canadian search result`.
- Prefer location-aware next steps when the target is a person, phone number, company, or municipality.

## Quality Bar

- Do not overstate Canadian linkage from a generic search hit alone.
- Do not mix federal, provincial, and private-directory results without labeling them.
- Do not treat a Canada-focused search URL as evidence.
